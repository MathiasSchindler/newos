# [

## NAME

[ - evaluate a conditional expression (bracket form)

## SYNOPSIS

```
[ EXPRESSION ]
```

## DESCRIPTION

`[` is an alias for `test` that requires a closing `]` as its last argument.
It evaluates EXPRESSION and exits 0 (true) or 1 (false). See **test(1)** for
the full set of supported operators.

## CURRENT CAPABILITIES

Same as `test`. All file tests, string tests, integer comparisons, file
comparisons, and logical operators documented in `test(1)` are supported.
The closing `]` argument is required.

## OPTIONS

None.

## LIMITATIONS

- Same as `test`.
- The closing `]` must be a separate argument, not attached to the preceding
  token.
- It is a command form only; shell-specific parsing of `[[ ... ]]`, pattern
  matching, and short-circuit expression syntax belongs to the shell and is not
  implemented here.

## EXAMPLES

```
[ -f /etc/passwd ] && echo "exists"
[ "$a" = "$b" ] || echo "not equal"
[ -d src ] || mkdir src
[ "$x" -gt 0 ]
```
